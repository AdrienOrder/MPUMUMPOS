# MPUMUMPOS - Мобильное приложение метеостанции

## 1. Общая архитектура приложения

Приложение построено по принципу разделения ответственности. Выделяются следующие уровни:

- Уровень пользовательского интерфейса (UI) - экраны, с которыми взаимодействует пользователь.
- Уровень бизнес-логики - управление Bluetooth-соединением, обработка команд, парсинг данных, логика работы с хранилищем файлов.
- Уровень данных - локальное хранилище (база данных и файловая система) для сохранения информации об устройствах и загруженных CSV-файлах.
- Уровень связи - модуль для работы с Bluetooth-модулем HC-05 (отправка/приём команд, чтение потоковых данных).

Все экраны связаны через навигационное меню (нижняя панель), а также через явные переходы по нажатию на элементы.

## 2. Описание экранов и их функций

### 2.1. Главный экран (MainActivity)

Назначение: подключение к Bluetooth-устройству, просмотр и изменение его рабочих параметров.

Элементы и функции:

- Индикатор подключения (статус, имя устройства).
- Кнопка подключения/отключения.
- Поля параметров (интервал, время старта, текущее время устройства) - только для чтения.
- Режим редактирования - по нажатию поля становятся активными, после подтверждения новые значения отправляются на устройство.
- Кнопка "Синхронизировать текущее время" - отправляет на устройство системное время телефона.
- Кнопка "Опрос данных с устройства" - запрашивает текущие значения интервала, времени старта и времени устройства.

### 2.2. Экран логов (LogsActivity)

Назначение: отображение истории событий, обмена командами и ошибок.

Функции:

- Прокручиваемый список логов (время события + текст).
- Кнопка "Очистить логи" - удаляет все записи из списка.
- Кнопка "Скопировать логи" - копирует всё содержимое в буфер обмена.

### 2.3. Экран выбора Bluetooth-устройства (DeviceListActivity)

Назначение: выбор из списка ранее сопряжённых Bluetooth-адаптеров.

Функции:

- Отображение списка сопряжённых устройств (имя, MAC-адрес).
- Обработка нажатия - подключение и возврат на главный экран.
- Запрос разрешений на Bluetooth и местоположение (Android 12+).
- Обработка ошибок (Bluetooth выключен, нет устройств).

### 2.4. Экран "Хранилище" (StorageActivity)

Назначение: доступ ко всем устройствам и сохранённым CSV-файлам. Построен по принципу "дерево".

Функции:

- Список устройств - имя, MAC-адрес, количество загруженных CSV-файлов.
- Удаление устройства - диалог подтверждения; при удалении удаляется запись и все связанные файлы.
- Переход к файлам - при нажатии на устройство открывается экран со списком CSV-файлов.

### 2.5. Экран списка CSV-файлов (FileListActivity)

Назначение: показать все загруженные с конкретного устройства файлы и предоставить действия с ними.

Функции:

- Список файлов - имя файла, дата и время загрузки, размер.
- Действие "Скачать" - копирует CSV-файл в папку "Загрузки" Android.
- Действие "Визуализировать" - переход на экран построения графиков.

### 2.6. Экран визуализации (VisualizationActivity)

Назначение: выбор параметров из CSV-файла и построение графика зависимости от времени.

Функции:

- Выбор измерений - автоматически определяются колонки с данными из CSV-файла.
- Множественный выбор параметров (через чекбоксы).
- Построение графика - ось X (дата и время), ось Y (значения).
- Несколько линий разными цветами с легендой.
- Обработка пропусков - "(NO REPLY)" или пустое значение не рисуется.
- Масштабирование и прокрутка.
- Ограничение по оси X (диапазон дат).

### 2.7. Экран графиков (ChartActivity)

Назначение: отображение графиков с использованием библиотеки MPAndroidChart.

## 3. Взаимодействие экранов и навигация

- Нижнее навигационное меню содержит три пункта: "Главный", "Логи", "Хранилище".
- Из главного экрана возможен переход к экрану выбора Bluetooth-устройства.
- Из хранилища через нажатие на устройство осуществляется переход к списку его CSV-файлов.
- Из списка файлов - к экрану визуализации.
- Из визуализации можно вернуться назад.

## 4. Хранение данных

- База данных SQLite хранит:
  - Таблицу устройств (имя, MAC-адрес, дата первого подключения, дата последней активности).
  - Таблицу CSV-файлов (имя файла, путь, дата загрузки, размер, идентификатор устройства).
- Файловая система - CSV-файлы сохраняются в закрытой директории приложения.
- При удалении устройства все его файлы удаляются.

## 5. Формат CSV данных

Файлы содержат данные с метеостанции в следующем формате:

### Параметры

- TempAir - Темп��ратура воздуха (C)
- HumAir - Влажность воздуха (%)
- PressAir - Атмосферное давление (hPa)
- TempSoil - Температура почвы (C)
- MoinsSoil - Влажность почвы (%)
- WindSpeed - Скорость ветра (м/с)
- WindGust - Порыв ветра (м/с)
- WindDirect - Направление ветра (градусы)
- Solar - Солнечная радиация (Вт/м2)

### Особенности

- Пропуски данных обозначаются как "-" или "(NO REPLY)"
- Графики строятся только для валидных значений
- Поддерживается фильтрация по диапазону дат

## 6. Импорт CSV файлов

### Способ 1: Из внутреннего хранилища

- Файлы, загруженные с Bluetooth-устройства, сохраняются в хранилище приложения
- Доступны через экран "Хранилище" -> выбрать устройство -> выбрать файл

### Способ 2: Из папки "Загрузки" Android

- Пользователь может импортировать CSV-файлы из стандартной папки "Загрузки" (Downloads)
- Для этого нужно перейти в "Хранилище" и выбрать опцию импорта
- Импортированные файлы ассоциируются с dummy-устройством "Imported"
- Поддерживаются файлы, соответствующие формату данных метеостанции
